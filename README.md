# Заявка на согласование/изменение договорных документов

## Правило (кодировка скриптов)
Всегда создаем скрипты на английском, чтобы не було проблем с кодировкой. Не используем кирилицу в скриптах.

## 📋 Описание

HTML-форма для создания заявок на согласование или изменение договорных документов. Предназначена для внутреннего использования в компании БСО.

## ✨ Функциональность

### Основные возможности:
- **Автоматическая генерация номера заявки** — формат: `ДДММГГ_Фамилия_БСО_№`
- **Экспорт в PDF** — генерация готового документа для скачивания
- **Валидация полей** — проверка обязательных полей перед отправкой
- **Адаптивный дизайн** — корректное отображение на мобильных устройствах

### Типы заявок:
- Изменение договорных документов
- Расторжение договорных документов

### Типы контрагентов:
| Тип | Дополнительные поля |
|-----|---------------------|
| **Подрядчик** | Проект, адрес, вид работ, стоимость, аванс, график платежей, гарантии, сроки |
| **Поставщик** | Проект, адрес, номенклатура, сумма поставки |
| **Заказчик** | Проект, адрес объекта |

### Виды договорных документов:
- Договор
- Дополнительное соглашение
- Прочие документы

## 🛠 Технологии

- **HTML5** — структура страницы
- **CSS3** — стилизация и адаптивность
- **JavaScript (Vanilla)** — логика формы и валидация
- **html2pdf.js** — генерация PDF-документов

## 📁 Структура полей формы

```
├── Номер заявки (автогенерация)
├── Инициатор заявки (ФИО)
├── Дата заявки
├── Тип заявки (Изменение/Расторжение)
├── Информация о документе
│   ├── Вид договорного документа
│   ├── Наименование документа
│   └── Содержание документа
├── Контрагент
│   ├── Наименование
│   ├── Контактное лицо
│   ├── Телефон (+7 формат)
│   ├── Email
│   ├── Наличие уставных документов
│   └── Дата предоставления (если нет документов)
├── ЭДО
│   ├── Возможность работы с ЭДО
│   └── Наименование ЭДО-системы
├── Детали по типу контрагента (условные поля)
└── Особые условия по договору
```

## 🎨 Особенности интерфейса

- **Условные блоки** — поля появляются в зависимости от выбора пользователя
- **Модальное окно** — уведомление об успешном создании заявки
- **Подсветка ошибок** — визуальная индикация незаполненных обязательных полей
- **Автоформатирование телефона** — автоматическое добавление префикса +7

## 💾 Хранение данных

- Счётчик заявок сохраняется в `localStorage` браузера
- PDF-файл скачивается на устройство пользователя
- Данные формы не отправляются на сервер (offline-режим)

## 🚀 Использование

1. Откройте файл `Заявка16.html` в браузере
2. Заполните все обязательные поля (отмечены *)
3. Нажмите "Сформировать заявку"
4. Сохраните сгенерированный PDF-файл

## 📱 Адаптивность

Форма адаптирована для устройств с шириной экрана менее 768px:
- Увеличенный размер шрифта в полях ввода
- Адаптивные отступы контейнера
